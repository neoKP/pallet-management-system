## 5. การบันทึก Movement (รับ-จ่าย)

### 5.1 ขั้นตอนการบันทึกรับพาเลทเข้า (IN)

1. เลือก Tab **"Movement"**
2. เลือกประเภท **"IN (รับเข้า)"**
3. เลือก **แหล่งที่มา** (Source):
   - สาขาอื่นในเครือ
   - คู่ค้าภายนอก (Neo Corp, Sino, etc.)
4. เลือก **ประเภทพาเลท**
5. ระบุ **จำนวน**
6. กรอกข้อมูลเพิ่มเติม (ถ้ามี):
   - เลขที่อ้างอิง (ECD)
   - ทะเบียนรถ
   - ชื่อคนขับ
   - ประเภทรถ
7. กดปุ่ม **"บันทึก"**

### 5.2 ขั้นตอนการบันทึกจ่ายพาเลทออก (OUT)

1. เลือก Tab **"Movement"**
2. เลือกประเภท **"OUT (จ่ายออก)"**
3. เลือก **ปลายทาง** (Destination):
   - สาขาอื่นในเครือ
   - คู่ค้าภายนอก
4. เลือก **ประเภทพาเลท**
5. ระบุ **จำนวน**
6. กรอกข้อมูลเพิ่มเติม
7. กดปุ่ม **"บันทึก"**

### 5.3 ประวัติการเคลื่อนไหว

ดูประวัติทั้งหมดได้ที่ส่วน **Transaction History** ด้านล่างของหน้า Movement

| คอลัมน์ | คำอธิบาย |
| :--- | :--- |
| วันที่ | วัน-เวลาที่บันทึก |
| ประเภท | IN / OUT / TRANSFER |
| ต้นทาง-ปลายทาง | สาขา/คู่ค้าที่เกี่ยวข้อง |
| พาเลท | ประเภทและจำนวน |
| สถานะ | PENDING / COMPLETED |
| เอกสาร | หมายเลขเอกสารอ้างอิง |

## 7. การจัดการ Pending Transfers

### 7.1 Pending Transfers คืออะไร

เมื่อสาขาหนึ่งส่งพาเลทไปยังอีกสาขาหนึ่ง:

1. สาขาต้นทางบันทึก **OUT** → สถานะ **PENDING**
2. สาขาปลายทางต้อง **ยืนยันรับ** → สถานะ **COMPLETED**
3. หลังยืนยัน → สต็อกสาขาปลายทางเพิ่มขึ้น

### 7.2 การยืนยันรับพาเลท

1. ไปที่ Tab **"Pending"**
2. ดูรายการที่รอยืนยัน
3. ตรวจสอบข้อมูล:
   - ต้นทาง
   - ประเภทพาเลท
   - จำนวน
4. กดปุ่ม **"ยืนยันรับ"** หรือ **"ไม่รับ (Reject)"**

### 7.3 การแก้ไขจำนวน (Split)

หากได้รับพาเลทไม่ตรงตามที่ส่งมา:

1. กดปุ่ม **"แก้ไข"** ที่รายการ
2. ระบุจำนวนที่ได้รับจริง
3. ระบบจะปรับยอดให้อัตโนมัติ

## 8. ระบบซ่อมบำรุง (Maintenance/DSS)

### 8.1 ภาพรวม DSS

DSS (Decision Support System) รองรับ:

- การซ่อมแซมพาเลท (Repair)
- การแปลงสภาพพาเลท (Convert)
- การตัดทิ้ง (Scrap/Discard)

### 8.2 การรับพาเลทเข้าซ่อม (Repair Inbound)

1. ไปที่ Tab **"Maintenance"** → **"รับเข้าซ่อม"**
2. เลือกประเภทพาเลท
3. ระบุจำนวนที่รับเข้าซ่อม
4. กดปุ่ม **"บันทึก"**
5. พาเลทจะถูกย้ายไปยัง **คลังซ่อมบำรุง (maintenance_stock)**

### 8.3 การดำเนินการซ่อม (Repair Process)

1. ไปที่ Tab **"Maintenance"** → **"ดำเนินการซ่อม"**
2. เลือกรายการจากคลังซ่อม
3. ระบุผลการซ่อม:
   - **จำนวนที่ซ่อมได้** (Fixed)
   - **จำนวนที่ต้องตัดทิ้ง** (Scrapped)
   - **ประเภทพาเลทผลลัพธ์** (Target Pallet)
4. กดปุ่ม **"ประมวลผล"**

### 8.4 รายงาน Yield

ระบบคำนวณ **Yield** อัตโนมัติ:

```text
Yield = (จำนวนที่ซ่อมได้ / จำนวนที่รับเข้าซ่อม) × 100%
```

### 8.5 ตารางรายงานการตัดทิ้ง

ดูประวัติการตัดทิ้งได้ที่ **"Scrapped Report"**:

- วันที่ตัดทิ้ง
- ประเภทพาเลท
- จำนวน
- เหตุผล
